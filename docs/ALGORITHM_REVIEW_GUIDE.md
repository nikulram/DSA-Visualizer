# Algorithm Review Guide

Every algorithm should be reviewed for correctness and visualization quality.

## Correctness Checklist

```text
- Does the trace follow the real algorithm?
- Are edge cases handled?
- Are step messages accurate?
- Do visual state fields make sense?
- Does the final state clearly show completion?
```

## Visual Checklist

```text
- Is the active element highlighted?
- Is selected state visible?
- Is the explanation useful?
- Is input editable where appropriate?
- Does the visual work on smaller screens?
```

Required validation:

```bash
npm run check
```
